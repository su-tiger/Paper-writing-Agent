# 工作流可视化对比

## 1️⃣ 基础写作工作流（不带 RAG）

```
┌─────────────────────────────────────────────────────────────┐
│                        用户输入主题                          │
│                   "人工智能对教育的影响"                      │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
                  ┌──────────────┐
                  │   Planner    │  生成文章大纲
                  │  (规划节点)   │  - 引言
                  └──────┬───────┘  - 主体
                         │          - 结论
                         ▼
                  ┌──────────────┐
                  │   Dispatch   │  分发到多个 Writer
                  └──────┬───────┘
                         │
              ┌──────────┴──────────┐
              ▼                     ▼
       ┌─────────────┐       ┌─────────────┐
       │  Writer A   │       │  Writer B   │
       │ (学术风格)   │       │ (批判风格)   │
       └──────┬──────┘       └──────┬──────┘
              │                     │
              └──────────┬──────────┘
                         ▼
                  ┌──────────────┐
                  │   Reviewer   │  审核文章质量
                  │  (审核节点)   │  - 结构是否合理？
                  └──────┬───────┘  - 内容是否充实？
                         │
                    ┌────┴────┐
                    │  判断    │
                    └────┬────┘
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │ 通过审核 │    │修改大纲  │    │修改正文  │
    │  结束   │    │返回Planner│   │返回Writer│
    └─────────┘    └─────────┘    └─────────┘
```

**特点：**
- 纯 LLM 生成，不依赖外部知识
- 适合创意写作、观点文章
- 可能产生幻觉或不准确信息

## 2️⃣ RAG 写作工作流（带检索）

```
┌─────────────────────────────────────────────────────────────┐
│                        用户输入主题                          │
│              "深度学习在 NLP 中的应用"                        │
└────────────────────────┬────────────────────────────────────┘
                         │
                         ▼
                  ┌──────────────┐
                  │  Retriever   │  从向量数据库检索
                  │  (检索节点)   │  
                  └──────┬───────┘
                         │
                         ▼
              ┌──────────────────────┐
              │   检索到的文档片段     │
              │  📄 BERT 模型介绍     │
              │  📄 GPT 系列模型      │
              │  📄 Transformer 架构  │
              └──────────┬───────────┘
                         │
                         ▼
                  ┌──────────────┐
                  │ Summarizer   │  基于检索结果生成综述
                  │ (总结节点)    │  - 整合多个文档
                  └──────┬───────┘  - 提炼核心观点
                         │          - 形成连贯叙述
                         ▼
                  ┌──────────────┐
                  │   Reviewer   │  审核综述质量
                  │  (审核节点)   │  - 内容完整性 30%
                  └──────┬───────┘  - 逻辑性 25%
                         │          - 学术性 20%
                         │          - 创新性 15%
                         │          - 可读性 10%
                    ┌────┴────┐
                    │  判断    │
                    └────┬────┘
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
    ┌─────────┐    ┌─────────┐    ┌─────────┐
    │评分 ≥ 8  │    │评分 < 8  │    │达到最大  │
    │ 通过审核 │    │需要优化  │    │迭代次数  │
    │  结束   │    │   ↓     │    │  结束   │
    └─────────┘    └────┬────┘    └─────────┘
                        │
                        ▼
                  ┌──────────────┐
                  │   Refiner    │  根据反馈优化
                  │  (优化节点)   │  - 改进逻辑
                  └──────┬───────┘  - 补充内容
                         │          - 提升学术性
                         │
                         └──────────┐
                                    │
                         ┌──────────┘
                         ▼
                  ┌──────────────┐
                  │   Reviewer   │  重新审核
                  │  (再次审核)   │
                  └──────────────┘
                         │
                    (循环直到通过)
```

**特点：**
- 基于真实文档生成，准确性高
- 适合学术综述、技术报告
- 可以标注引用来源
- 支持最新知识（只要更新向量库）

## 3️⃣ 关键区别对比

### 数据流对比

**基础工作流：**
```
主题 → LLM 内部知识 → 生成内容
```

**RAG 工作流：**
```
主题 → 向量检索 → 外部文档 → 基于文档生成
```

### 节点对比

| 阶段 | 基础工作流 | RAG 工作流 | 说明 |
|------|-----------|-----------|------|
| 第1步 | Planner | Retriever | RAG 先检索，不先规划 |
| 第2步 | Writer | Summarizer | RAG 是总结，不是创作 |
| 第3步 | Reviewer | Reviewer | 都需要审核 |
| 第4步 | (返回 Planner/Writer) | Refiner | RAG 有专门的优化节点 |

### 适用场景对比

```
┌─────────────────────┬──────────────┬──────────────┐
│      场景           │  基础工作流   │  RAG 工作流   │
├─────────────────────┼──────────────┼──────────────┤
│ 创意写作            │      ✅      │      ❌      │
│ 观点文章            │      ✅      │      ❌      │
│ 学术综述            │      ❌      │      ✅      │
│ 技术报告            │      ❌      │      ✅      │
│ 需要引用来源        │      ❌      │      ✅      │
│ 需要最新知识        │      ❌      │      ✅      │
│ 追求创意多样性      │      ✅      │      ❌      │
│ 追求事实准确性      │      ❌      │      ✅      │
└─────────────────────┴──────────────┴──────────────┘
```

## 💡 理解要点

1. **RAG 的核心**：先检索，再生成（Retrieve then Generate）
2. **迭代优化**：通过 Reviewer → Refiner 循环提升质量
3. **状态管理**：所有节点共享同一个状态对象
4. **条件跳转**：使用 Command 实现动态流程控制
5. **防止死循环**：设置最大迭代次数限制

这就是 RAG + Agent 的完整工作流程！🎉
